# KMS Core Concepts — Huawei Cloud Key Management Service

## 1. What Is KMS?

Huawei Cloud KMS provides HSM-backed encryption key management. A Customer Master Key (CMK)
is the top-level key that encrypts data keys (DEKs). CMKs never leave the HSM boundary.

| Concept | Description |
|---|---|
| CMK (主密钥) | Top-level key managed by KMS; never exported in raw form |
| DEK (数据密钥) | Data Encryption Key; generated by KMS, used by client; can be exported |
| HSM | Hardware Security Module; FIPS 140-2 Level 2 certified; keys stored in HSM |
| Grant | Temporary permission delegation; can be revoked |
| BYOK | Bring Your Own Key; import external key material |

## 2. Key Types

| Type | Code | Use Case | HSM |
|---|---|---|---|
| 对称默认密钥 | `SYMMETRIC_DEFAULT` | OBS, RDS, EVS encryption | Yes |
| 异步非对称 | `ASYMMETRIC_<ALG>_VERIFY` | Code signing, document signing | Yes |
| 异步非对称加密 | `ASYMMETRIC_<ALG>_ENCRYPT` | Key transport | Yes |

> Default: `SYMMETRIC_DEFAULT` for all data encryption use cases.

## 3. Key States

```
[CREATING] --HSM provisioned--> [ENABLED] <--disable-key--> [DISABLED]
                                                      |
                                          schedule-key-deletion
                                                      ↓
                                          [PENDING_DELETION] (7-1096 d)
                                                      |
                                                  window expires
                                                      ↓
                                                    [DELETED]
```

| State | Meaning | Operations Allowed |
|---|---|---|
| `ENABLED` | Normal operational state | All cryptographic operations |
| `DISABLED` | Temporarily disabled | None — dependent services fail |
| `PENDING_DELETION` | Deletion scheduled | None during window |
| `CREATING` | HSM provisioning in progress | None |

## 4. Key Rotation

- Symmetric keys: automatic rotation every 365 days (free).
- Rotation does **NOT** re-encrypt existing data — it rotates the key **wrapper** (DEK envelope).
- To re-encrypt data: decrypt with old key, re-encrypt with new key (agent must implement manually).

## 5. Grants vs IAM Policies

| Mechanism | Scope | Revocable | Use Case |
|---|---|---|---|
| Grant | Per key, per grantee | Yes (immediate) | Service-to-service delegation |
| IAM Policy | Account-wide | No (without policy update) | Admin access |
| RAM Role | Account-wide | No | Service-linked roles |

## 6. BYOK (Bring Your Own Key)

1. Create import-only CMK: `key_type = SYMMETRIC_IMPORT`
2. Download public key + import token (valid 24h)
3. Wrap key material externally with public key (RSAES-OAEP-SHA-256)
4. Import wrapped key material
5. Key becomes usable — key material never leaves customer's control during transport

## 7. Key Quotas (Dynamic)

```bash
# Show key count vs quota
hcloud kms list-keys --region {{env.HW_REGION_ID}} --output json | jq 'length'
```

> Per `AGENTS.md` TE-1, **never hardcode** quota numbers. Always query at run time.
> Default: 100 CMKs per account per region.
