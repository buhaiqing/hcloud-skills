# DNS Fault Knowledge Base

> 本知识库汇总 DNS 常见故障模式、根因定位速查与跨 skill 联动,基于 `monitoring.md` 中定义的真实指标信号。命令示例以 `hcloud` CLI 为主,不确定的参数以 `{{user.*}}` / `{{env.*}}` 占位。

---

## 常见故障模式

| 故障ID | 症状 | 根因 | 排查步骤 | 恢复动作 |
|---|---|---|---|---|
| DNS-01 | 解析成功率下降 | 解析服务器异常或线路故障 | 查解析成功率 `< 95%`;检查线路健康 | 切换解析线路;提工单 |
| DNS-02 | NXDOMAIN 激增 | 域名配置错误或扫描攻击 | 查 `nxdomain_count > 1000/min` | 核对域名配置;封禁扫描源 |
| DNS-03 | 解析延迟高 | 递归链路拥塞或上游慢 | 查 `p99 > 100ms`;追踪递归路径 | 优化解析线路;就近接入 |
| DNS-04 | TTL 风暴 | 同名高频刷新 | 查同名请求 `> 1000 req/s` | 收敛 TTL 策略;限制变更频率 |
| DNS-05 | NS 委托链不一致 | 父域/子域 NS 记录不匹配 | 核查 NS 委托链各层记录 | 修正 NS 记录保持一致 |
| DNS-06 | 解析被劫持 | 中间链路篡改 | 对比多节点解析结果 | 启用 DNSSEC;联动 Anti-DDoS |
| DNS-07 | 配额耗尽 | 解析请求超配额 | 查配额使用率 | 提升配额或扩容 |
| DNS-08 | 区域解析失败 | 单区域节点异常 | 按区域对比成功率 | 切流至健康节点 |

---

## 根因定位速查

- **成功率 < 95%** → 先区分线路故障还是域名配置错误(核对近期变更)。
- **NXDOMAIN 突增** → 多为扫描/攻击或域名误删,优先核对域名是否存在。
- **p99 延迟高** → 关注递归链路与上游权威服务器,而非仅看本地节点。
- **NS 委托链不一致** → 父域与子域 NS 必须完全一致,否则随机解析失败。
