# IAM Fault Knowledge Base

> 本知识库汇总 IAM 常见故障模式、根因定位速查与跨 skill 联动。信号提取自 `SKILL.md` 与 AIOps 模式。命令示例以 `hcloud` CLI 为主,不确定参数以 `{{user.*}}` / `{{env.*}}` 占位。

---

## 常见故障模式

| 故障ID | 症状 | 根因 | 排查步骤 | 恢复动作 |
|---|---|---|---|---|
| IAM-01 | 权限拒绝 | 策略未授权 `AccessDenied` | 查报错中的 action/resource;核对策略 | 追加最小权限策略 |
| IAM-02 | AK/SK 失效或泄露 | 调用返回鉴权失败 | 查 AK 状态与最近调用来源 | 轮转 AK/SK;吊销泄露凭证 |
| IAM-03 | 策略配置错误 | 策略语法/作用域错误 | 校验策略 JSON 与绑定关系 | 修正策略并重新绑定 |
| IAM-04 | MFA 失效 | 二次验证失败 | 查 MFA 设备状态与绑定 | 解绑重绑 MFA;应急放行 |
| IAM-05 | 配额耗尽 | 用户/策略/项目数达上限 | 查配额使用率 | 提升配额或清理闲置 |
| IAM-06 | 跨账号委派异常 | 委托关系失效 | 查委托协议与信任策略 | 重建委托;校验信任方 |
| IAM-07 | 角色临时凭证过期 | STS token 失效 | 查 token 有效期 | 重新申请临时凭证 |
| IAM-08 | 权限放大风险 | 策略过于宽松 | 审计策略最小权限 | 收敛权限至最小集 |

---

## 根因定位速查

- **AccessDenied** → 区分"完全无策略"还是"策略条件不满足"(IP/时间/MFA 条件)。
- **AK 失效** → 优先判断是否泄露(异常调用来源),泄露则立即吊销而非仅轮转。
- **跨账号异常** → 多为信任策略中 Principal 配置错误,核对双方账号 ID。
- **配额耗尽** → 多为测试/闲置实体堆积,清理优于直接提额。

## 跨 skill 联动

| 故障场景 | 联动 Skill | 目的 |
|---|---|---|
| 任意 product-ops 鉴权失败 | 各 `huaweicloud-*-ops` | 提供正确权限策略 |
| 操作审计追溯 | `huaweicloud-cts-ops` | 查审计日志定位异常操作 |
| 费用/配额 | `huaweicloud-billing-ops` | 配额与计费核对 |
| 凭证泄露应急 | `huaweicloud-hss-ops` | 主机侧排查泄露源 |
